# Financial Year And Periods

## النموذج

### FinancialYear

يمثل سنة مالية صريحة داخل الشركة ويحتوي على:

- `Code`
- `Name`
- `StartDate`
- `EndDate`
- `IsActive`

القواعد:

- لا يسمح بتداخل السنوات داخل نفس الشركة
- توجد سنة نشطة واحدة فقط

### FinancialPeriod

يمثل فترة محاسبية داخل سنة مالية، ويحتوي على:

- `FinancialYearId`
- `Code`
- `Name`
- `StartDate`
- `EndDate`
- `Status`
- `IsAdjustmentPeriod`

القواعد:

- كل فترة تنتمي إلى سنة مالية واحدة
- لا يسمح بتداخل الفترات داخل نفس السنة
- الفترات تغطي نطاق السنة نفسها

## كيف يتم التوليد

عند إنشاء سنة مالية، يتم توليد فترات شهرية تلقائيًا.

إذا بدأت السنة أو انتهت في منتصف الشهر، يتم قص أول وآخر فترة لتطابق حدود السنة.

مثال:

- سنة من `2026-03-15` إلى `2026-05-10`
- الفترات ستكون:
  - `2026-03-15 -> 2026-03-31`
  - `2026-04-01 -> 2026-04-30`
  - `2026-05-01 -> 2026-05-10`

## حالات الفترة

- `Open`
- `Closed`

الفترة المقفلة تمنع:

- إنشاء قيد جديد داخلها
- تعديل قيد بتاريخ داخلها
- ترحيل مسودة بتاريخ داخلها

## كيف يتم التحقق عند الترحيل

كل مسار ترحيل أو إنشاء/تعديل للقيد يمر عبر `FinancialPeriodGuard`.

التحقق يتم بهذا الترتيب:

1. هل توجد سنة مالية نشطة تغطي تاريخ القيد؟
2. هل توجد فترة مالية تحتوي التاريخ؟
3. هل الفترة مفتوحة؟
4. هل التاريخ بعد `LockedThroughDate`؟

إذا فشل أي شرط، يرفض النظام العملية برسالة عمل واضحة.

## العلاقة مع البيانات القديمة

إذا كانت هناك قيود تاريخية أقدم من تعريف السنوات والفترات، تقوم مرحلة bootstrap ب:

- إنشاء سنوات مالية مفقودة عند الحاجة
- إنشاء فتراتها الشهرية
- ربط القيود القديمة بالسنة والفترة المناسبتين
